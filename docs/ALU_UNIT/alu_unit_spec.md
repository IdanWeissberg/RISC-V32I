# ALU Unit Specification (RV32I)

## 1. Overview
The **ALU (Arithmetic Logic Unit)** is the primary combinatorial execution block of the processor. It performs 32-bit arithmetic, bitwise logic, and shifting operations. The specific operation is selected via a 4-bit control signal (`alu_op_in`) generated by the ALU Control unit.

## 2. Design Hierarchy
* **Type**: Purely Combinatorial.
* **Inputs**: Two 32-bit data operands, one 4-bit control signal.
* **Outputs**: One 32-bit result.
* **Placement**: Execute (EX) Stage.

## 3. Port Map (Interface)

| Port Name      | Direction | Width | Type  | Description                                                                 |
| :------------- | :-------- | :---- | :---- | :-------------------------------------------------------------------------- |
| `read_data_1`  | Input     | 32    | Wire  | First operand (rs1).                                                        |
| `alu_mux`      | Input     | 32    | Wire  | Second operand (rs2 or Immediate).                                          |
| `alu_op_in`    | Input     | 4     | Wire  | 4-bit operation selector.                                                   |
| `alu_result`   | Output    | 32    | Reg   | 32-bit computation output.                                                  |

## 4. Operational Logic
The unit evaluates the `alu_op_in` signal within an `always @(*)` block. 

### Arithmetic & Logic:
* **ADD/SUB**: Performs 32-bit addition or subtraction. Note: RISC-V ignores integer overflow.
* **AND/OR/XOR**: Standard bitwise operations between `read_data_1` and `alu_mux`.
* **LUI (Pass-through)**: Bypasses `read_data_1` and outputs `alu_mux` directly (used for Loading Upper Immediate).

### Shifting (RV32I Standard):
* **Amount Masking**: Only the 5 least significant bits of the second operand (`alu_mux[4:0]`) are used as the shift amount (supporting shifts from 0 to 31).
* **SLL/SRL**: Logical shifts (left/right) filling vacated bits with zeros.
* **SRA (Arithmetic)**: Uses `$signed` type-casting to ensure the sign bit (bit 31) is propagated during the right shift.

### Comparisons:
* **SLT (Signed)**: Returns `32'b1` if `read_data_1 < alu_mux` (as signed integers), else `32'b0`.
* **SLTU (Unsigned)**: Returns `32'b1` if `read_data_1 < alu_mux` (as unsigned integers), else `32'b0`.

## 5. Implementation Safeguards
* **Latch Prevention**: The module uses a **Pre-assignment Strategy**. At the start of the procedural block, `alu_result` is initialized to `32'b0`. This ensures all execution paths have a defined output, preventing the inference of transparent latches during synthesis.
* **Signed Arithmetic**: Employs Verilog system functions (`$signed`, `$unsigned`) for clear and synthesis-safe signed comparisons and shifts.

## 6. Timing
* **Combinatorial Delay**: The output reacts immediately to any change in inputs. In the full processor timing closure, this path (from RegFile through ALU to Memory/RegFile) usually forms the critical path.
